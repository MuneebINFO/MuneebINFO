## What I build

**Algorithms & data structures**
Implementations that go beyond textbook exercises — visualized, benchmarked, and compared across complexity classes. I enjoy translating theoretical analysis into working, measurable code. This includes string algorithms (palindrome detection with naïve, center-expansion, and Manacher approaches), sorting, and complexity benchmarking across real hardware.

**Networked systems & infrastructure**
Client-server applications, VPN deployment, socket programming. Projects that require thinking about protocol design, concurrency, and real-world constraints. This includes building a full TCP-based chat application and deploying OpenVPN on a real network setup.

**Game development**
Interactive projects built with Godot — used as a medium to demonstrate CS concepts visually. From a multiplayer Tetris variant to algorithm visualizers, games are a great forcing function for clean architecture and real-time performance.

**Optimization & graph algorithms**
Pathfinding, combinatorial problems, heuristics. Projects that involve modeling a real-world problem as a graph or optimization task — including shortest-path on real map data (Dijkstra) and approximate solutions to NP-hard problems like the Travelling Salesman Problem.

**Full-stack & SaaS exploration**
Web applications with modern stacks. I'm interested in how software gets built *and* shipped — product design, tech stack choices, and business viability included. Currently exploring ESG/compliance automation as a SaaS opportunity for SMEs.
